## Critical Anti-Patterns (do NOT repeat these)

| Pattern | Description | Severity | Prevention |
|---------|-------------|----------|------------|
| **Module-list lag** | When moving a helper that imports path constants (`STACKS_PATH`, `DATA_DIR`, `DASHBOARDS_DIR` etc) between modules, the `_MODULES` list in `planner/engine/_root_patch.py` must be updated. Otherwise `maybe_patch_root(data_root)` silently fails to patch the new module and tests using `tmp_path` get wrong data. Already cost one failed test cycle in A2. | advisory | When adding a new module that does `from planner.io import …`, add it to `_root_patch._MODULES`. Already done for `relations_surreal` and `audit_surreal`. |
| **Panel framing bias** | When asking the expert panel about replacing/porting an existing function, framing matters. First two rounds on `collect_similar_substances` implicitly treated Python as "the thing being replaced", which biased recommendations toward porting. User had to explicitly correct: "what if current Python is actually the best fit?". Third round (correctly framed as peer comparison) reversed the verdict. | advisory | Frame panel discussions as peer comparison including the status quo, not as "evaluate replacement options". Don't assume the user wants a change just because they invoked a replacement-shaped question. |
