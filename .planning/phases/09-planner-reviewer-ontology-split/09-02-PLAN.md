<objective>
Replace the substance card's flat-namespace schema with the v2 nested
`schedule:` / `knowledge:` shape, update the `Substance` dataclass to mirror
the split, make `load_substance` a dual-format reader, and update every
downstream consumer that reads per-namespace substance fields.

Purpose: This plan completes the Planner/Reviewer separation at the code
boundary. The Planner-relevant fields (`intake`, `timing`, `activity`,
`prefer_with`) become structurally separate from the Reviewer-relevant fields
(`is_`, `effect`, `risk`, `dashboard`, `pathway`). The scheduler's trait
aggregation drops `risk:` and `effect:` (the three scheduling effect slugs
now arrive via `timing:`). Plan 03 follows immediately with the 198-card
migration script.

Output: Schema, contracts, loader, scheduler trait aggregator, audit
cleanup-section iterator, dashboards resolver, and `cmd_review_substance`
display loop — all aligned to the v2 Substance shape.
</objective>
