<objective>
Foundation step: relocate the three scheduling-relevant effect slugs to a new `timing:`
namespace in `data/traits.yaml`, bootstrap an empty `pathway:` namespace, retire the
`separate_from:` mechanism on trait definitions, and update the planner-side
namespace registry and traits loader to match.

Purpose: This is wave 1 of the Phase 9 atomic migration. The substance loader is
NOT touched in this plan — it continues to read flat substance cards which still
carry `effect: [sleep_support|sleep_disruptive|energy_like]`. After this plan
lands, `planner check` will report unknown trait errors for any flat substance
card that references `effect:<timing slug>` (because the `effect:` namespace is
removed from the registry and those slugs now live under `timing:`). That is
EXPECTED and is the trigger for the wave 2 substance schema/contract/loader
overhaul — the migration must complete in one continuous executor session
before committing.

Output: An updated `traits.yaml`, `traits.schema.json`, `io.py`, and
`planner/cards/traits.py` that establish the v2 namespace topology. No
substance card changes in this plan.
</objective>
