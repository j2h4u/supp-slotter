<objective>
Replace the retired per-trait `separate_from:` mechanism with class-level
`competes` entries in `relations.yaml`. The Planner reads `knowledge.is:`
on active substances (the single documented exception to Planner ↛ knowledge
isolation) to resolve class membership, then blocks slot co-placement when a
class-level competes pair is present.

Purpose: This is the final correctness piece of the Planner/Reviewer split.
After plan 02, `must_separate` was already a no-op (no trait carried a
`separate_from:` field). This plan retires the dead code paths, threads a
new class-level competes mechanism through the scheduler, and seeds the
canonical `mineral ↔ fat_soluble` rule. The narrow `is:` exception is wired
explicitly so future readers see the design boundary in code.

Output: Relations schema + Relation dataclass + relations loader + scheduler
block-check rewrite + traits loader cleanup + traits.yaml class-level
competes seed + tests.
</objective>

Target _slot_is_blocked replaces must_separate with _class_level_competes_blocked:
```
def _class_level_competes_blocked(item_classes, existing_classes, class_competes_relations):
    for rel in class_competes_relations:
        src, tgt = rel.source_class, rel.target_class
        if src and tgt:
            if (src in item_classes and tgt in existing_classes) or \
               (tgt in item_classes and src in existing_classes):
                return True
    return False
```

item_classes = {
