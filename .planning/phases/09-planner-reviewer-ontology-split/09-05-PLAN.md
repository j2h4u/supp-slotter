<objective>
Introduce the new `planner review` command (the Reviewer entrypoint) that
surfaces concerns, relations status, risk flags, pathways, and dashboard
membership from the `knowledge:` section of substance cards. Slim
`planner audit` to its remaining role (cleanup candidates + optional
`--full` card-quality checks). Wire the new command through the engine
package, the CLI dispatcher, and the SKILL.md documentation.

Purpose: This is the final piece of the Planner/Reviewer split — the two
actors now have separate command surfaces. After this plan: `planner
schedule` (or the default command) produces slot assignments; `planner
review` produces all advisory content; `planner audit` is reduced to
structural-hygiene candidates only. The user (and the smart agent that
consumes planner output) interact with two distinct, predictable surfaces.

Output: New `cmd_review` function, slimmed `cmd_audit`, CLI dispatch
update, SKILL.md documentation, and a new test module pinning the command
behavior.
</objective>
