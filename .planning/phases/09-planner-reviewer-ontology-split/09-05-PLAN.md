<objective>
Introduce the new `planner review` command (the Reviewer entrypoint) that
surfaces concerns, relations status, risk flags, pathways, and dashboard
membership from the `knowledge:` section of substance cards. Slim
`planner audit` to its remaining role (cleanup candidates + optional
`--full` card-quality checks). Wire the new command through the engine
package, the CLI dispatcher, and the SKILL.md documentation.

THEN (Task 4 cleanup, per-09-REVIEWS.md MEDIUM-2 + SHOULD-fix 2): delete the
v1 loader fallback branch from `load_substance` (now dead code after plan 03
migrated all 198 cards) AND tighten `schema/substance.schema.json` to v2-only
by removing the transitional `oneOf` plus the `$defs.v1_flat` branch. Add
the flat-form schema rejection tests that were deferred from plan 03.
Coupling these two operations in a single commit prevents the v1 fallback
from becoming silently dead code that hides regressions.

Purpose: This is the final piece of the Planner/Reviewer split — the two
actors now have separate command surfaces, the v1 transition path is
removed, and the schema enforces the final v2 contract. After this plan:
`planner schedule` (or the default command) produces slot assignments;
`planner review` produces all advisory content; `planner audit` is reduced
to structural-hygiene candidates only. The user (and the smart agent that
consumes planner output) interact with two distinct, predictable surfaces.

Output: New `cmd_review` function, slimmed `cmd_audit`, CLI dispatch
update, SKILL.md documentation, new test module pinning the command
behavior, AND the v1-fallback removal + schema tightening cleanup.
</objective>
