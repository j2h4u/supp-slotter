<task type="auto">
  <name>Task 1: Create strict Substance/Product/Inventory schemas</name>
  <read_first>
    - schema/product.schema.json
    - schema/inventory.schema.json
    - schema/slots.schema.json
    - schema/traits.schema.json
    - .planning/phases/02-substance-product-yaml-model-split/02-CONTEXT.md
  </read_first>
  <files>schema/substance.schema.json, schema/product.schema.json, schema/inventory.schema.json, schema/slots.schema.json, schema/traits.schema.json</files>
  <action>
Add `schema/substance.schema.json` for universal cards with required `id`, `name`, and `traits`; optional `notes`, `unmatched_concerns`, and `prefer_with`; trait ids must keep the `namespace:name` pattern.

Rewrite `schema/product.schema.json` so products are concrete formulas with required `id`, `name`, and `components`. `components` must be a non-empty array of objects with required `substance`; optional `label`, `amount`, and `notes`; `substance` uses the same id pattern as card filenames. Product cards may have optional `notes`, `brand`, and `unmatched_concerns`, but must not have top-level universal `traits`.

Rewrite `schema/inventory.schema.json` so every `supplements.*` entry requires `product` and `stack`; allowed `stack` values stay `daily`, `training`, `inactive`; optional fields remain `dose`, `brand`, `notes`, and `traits_override`. `traits_override.add/remove` keep unique trait-id arrays.

Rewrite `schema/slots.schema.json` to allow only `label`, `order`, `stack`, `near`, and `food` on slots. `near` enum must be exactly `wake`, `breakfast`, `day_meal`, `sleep`, `workout_before`, `workout_after`; `food` is boolean; remove slot fields `time` and `activity`.

Extend `schema/traits.schema.json` only if needed so the existing trait structure can reference `near` and `food` match fields. Do not add MCP/import/database fields anywhere.
  </action>

Create concrete product formula cards in `data/products/*.yaml`. Every product card must have:

```yaml
id: <product_id>
name: "<concrete product/formula name>"
components:
  - substance: <substance_id>

```

For single-ingredient bottles, use one component that references the matching substance. For the current nattokinase shelf product, keep `id: nattokinase` as the inventory-facing product id and model components including:

```yaml

  - substance: nattokinase
  - substance: vitamin_b6
  - substance: vitamin_b12
```

Create `data/substances/vitamin_b6.yaml` and `data/substances/vitamin_b12.yaml` if they do not already exist. Put nattokinase empty-stomach/fibrinolytic facts on `data/substances/nattokinase.yaml`, not on the product formula.
