<task id="08-03-06">
  <title>DT-12f: Add Doctor Warning Playbook subsection</title>
  <action>
    In SKILL.md, add a new "### Doctor Warning Playbook" subsection after the Membership Flow section.

    Content:
    "### Doctor Warning Playbook

    WHEN to run uv run python -m planner doctor:
    - After any substance card edit (traits, dashboard: tags, is: tags)
    - After any dashboard yaml edit (from_traits changes, new cluster created)
    - After any data/traits.yaml change (new namespace entry, renamed slug)
    - Once at end of session before commit

    Per-warning-class resolution:

    dashboard.orphan_registration: A dashboard: slug is registered in data/traits.yaml but no substance card carries it under its dashboard: group.
    Causes: (A) trait registered for a planned cluster but substance tagging not done yet; (B) cluster abandoned but trait entry not removed.
    Resolution: (A) Tag the relevant substance cards with dashboard: <slug>. (B) Remove the trait entry from traits.yaml and the dashboard yaml file if it exists.

    dashboard.unused_trait: A dashboard: slug is carried by substance cards but no data/dashboards/*.yaml references it via from_traits.
    Causes: (A) dashboard yaml deleted while tags remained on substance cards; (B) dashboard yaml not yet created.
    Resolution: (A) Create data/dashboards/<slug>.yaml referencing from_traits: { dashboard: [<slug>] }, OR remove the tag from substance cards and the entry from traits.yaml. (B) Create the dashboard yaml.

    dashboard.slug_mismatch: A data/dashboards/<slug>.yaml exists without a matching dashboard:<slug> trait in traits.yaml, or vice versa.
    Canonical fix: (A) If yaml exists but trait missing — add the trait to traits.yaml under dashboard:. (B) If trait exists but yaml missing — create the dashboard yaml, or remove the trait entry.

    dashboard.empty_cluster: A dashboard's from_traits resolves to zero member substances.
    Causes: all tagged substances were removed; or from_traits slug does not match any substance's dashboard: list.
    Resolution: Tag substance cards with dashboard: <slug>, OR remove the dashboard yaml if the cluster is abandoned. If the cluster is an intentional placeholder for future use, add a notes: field explaining the intent to suppress the warning."
  </action>
