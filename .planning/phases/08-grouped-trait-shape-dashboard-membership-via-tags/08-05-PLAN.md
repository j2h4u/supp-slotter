# Plan 08-05: Stage 2d — Doctor Lifecycle Warnings (DT-14)

## Context

Stage 2 plan — independent, depends only on Stage 1 (08-01). Adds four new lifecycle warning classes to doctor.py. DT-14 is required (not optional): the primary consumer is agentic sessions with no persistent memory. Warnings must include actionable resolution text.

## Tasks

<task id="08-05-01">
  <title>DT-14: Implement four dashboard lifecycle warning classes in planner/engine/doctor.py</title>
  <action>
    In planner/engine/doctor.py, add four new lifecycle checks to collect_orphans(). These checks scan for mismatches between: traits.yaml dashboard: namespace entries, substance card dashboard: tags, and dashboard yaml from_traits references.

    Add a new helper function check_dashboard_lifecycle(trait_defs, substances, dashboard_files) -> dict[str, list[str]] that performs the four checks and returns warning messages keyed by warning class. Call it from collect_orphans() and merge its results into the returned sections dict.

    The four warning classes and their detection logic:

    **dashboard.orphan_registration** — trait registered but no substance carries it:
    - Collect all dashboard: slug names defined in trait_defs (namespace == "dashboard")
    - Collect all dashboard: slugs actually carried by substance cards (union of substance.dashboard tuples)
    - Registered but not carried = orphan_registration candidates
    - Message format: "Orphan registration: dashboard:{slug} defined in data/traits.yaml but no substance card has it under its dashboard: group. Likely cause: trait registered for a planned cluster but substance tagging not yet done. Resolution: tag relevant substance cards under dashboard:, OR remove the trait entry from data/traits.yaml if the cluster is abandoned."

    **dashboard.unused_trait** — substances carry it but no dashboard yaml references it:
    - Collect all dashboard: slugs carried by substance cards (union of substance.dashboard tuples)
    - Collect all dashboard: slugs referenced in dashboard yaml from_traits (dashboard namespace only)
    - Carried but not referenced = unused_trait candidates
    - Message format: "Unused trait: dashboard:{slug} is carried by {N} substance card(s) but no dashboard yaml references it via from_traits. Likely cause: dashboard yaml deleted while tags remained, OR yaml not yet created. Resolution: create data/dashboards/{slug}.yaml referencing it via from_traits: {{ dashboard: [{slug}] }}, OR remove the tag from substance cards and the entry from data/traits.yaml."

    **dashboard.slug_mismatch** — yaml file exists without matching trait, or trait exists without yaml:
    - Collect dashboard yaml stems: set of stems from DASHBOARDS_DIR.glob("*.yaml")
    - Collect dashboard trait slugs: set of short_name values where namespace == "dashboard" in trait_defs
    - yaml_stem not in trait_slugs → "yaml without trait" mismatch
    - trait_slug not in yaml_stems → "trait without yaml" mismatch
    - Message format (yaml without trait): "Slug mismatch: data/dashboards/{slug}.yaml exists but dashboard:{slug} is not registered in data/traits.yaml. Fix: add dashboard:{slug} entry to data/traits.yaml (with label and description)."
    - Message format (trait without yaml): "Slug mismatch: dashboard:{slug} is registered in data/traits.yaml but data/dashboards/{slug}.yaml does not exist. Fix: create data/dashboards/{slug}.yaml referencing from_traits: {{ dashboard: [{slug}] }}, or remove the trait entry from data/traits.yaml."

    **dashboard.empty_cluster** — from_traits resolves to zero member substances:
    - For each dashboard yaml, resolve membership using the same logic as build_dashboard_review() (scan substance cards for matching namespace/slug pairs)
    - Clusters resolving to 0 members (no substance matches any from_traits pair) = empty_cluster candidates
    - Message format: "Empty cluster: data/dashboards/{slug}.yaml from_traits resolves to zero member substances. Resolution: tag substances under dashboard: {slug}, OR remove the dashboard yaml if abandoned. (If this is an intentional placeholder, add a notes: field explaining the intent.)"

    Add these four sections to collect_orphans() return dict with keys:
    "dashboard.orphan_registration", "dashboard.unused_trait", "dashboard.slug_mismatch", "dashboard.empty_cluster"

    cmd_doctor() already iterates all sections and prints them — no changes needed to the display logic.

```
# Must exit 0

uv run pytest
# Must pass
```

To test warnings manually:
1. Add a dashboard: slug to traits.yaml without tagging any substance → expect orphan_registration
2. Tag a substance with a new dashboard: slug without creating yaml → expect unused_trait + slug_mismatch
3. Create a dashboard yaml without a traits.yaml entry → expect slug_mismatch
4. Create a dashboard yaml whose from_traits slug matches zero substances → expect empty_cluster
