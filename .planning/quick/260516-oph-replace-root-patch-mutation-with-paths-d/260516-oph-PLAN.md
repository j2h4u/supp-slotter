<objective>
Replace the module-level path-constant mutation pattern (`planner/engine/_root_patch.py` patches `DATA_DIR`, `STACKS_PATH`, etc. across a hardcoded `_MODULES` list at test time) with explicit `Paths` dataclass threading. After this plan: `_root_patch.py` is gone, the eight ROOT-derived path constants in `planner/io.py` are gone, every loader takes a `paths: Paths` parameter, and the only construction sites are inside the seven `cmd_*` entry points + `main()`.

Purpose: kill the hidden global mutation tested in CONCERNS.md "Global path patching for tests" — make data-root resolution explicit, type-checked, and impossible to forget. Tests drive `main(data_root=root)` instead of patching `sys.modules`.

Output: refactored planner package with `Paths` dataclass, deleted `_root_patch.py`, deleted module-level data-derived constants, updated test helper, byte-identical schedule.yaml.
</objective>

Phase A — define Paths:
1. In planner/io.py, add a `Paths` frozen dataclass with `slots=True` and fields: `root: Path`, `data: Path`, `substances: Path`, `products: Path`, `dashboards: Path`, `stacks_file: Path`, `relations_file: Path`, `schedule_file: Path`, `maintenance_lock: Path`. Implement two classmethod factories: `Paths.from_root(root: Path) -> Paths` (derives the eight subordinate paths from root, mirroring _root_patch._new_attrs) and `Paths.default() -> Paths` (returns `Paths.from_root(ROOT)`). Keep ROOT and SCHEMA_DIR as module-level. Re-export `Paths` in planner/io.py's public surface (no __all__ exists, but the symbol is importable).
2. Refactor `validate_schemas()` to take `paths: Paths` (required) and use `paths.data`, `paths.relations_file`, `paths.stacks_file`, `paths.substances`, `paths.products`, `paths.dashboards`.
