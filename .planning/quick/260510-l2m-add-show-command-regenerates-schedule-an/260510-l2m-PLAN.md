Schedule shape (relevant subset, produced by `_build_schedule_output` in `planner/engine/plan.py`):
```yaml
summary: {...}                # always present, non-empty dict
action_points: [...]          # list, possibly empty
review_contexts: [...]        # list, possibly empty
placement_notes: [...]        # list, possibly empty
pillboxes:                    # dict[str, Pillbox]
  daily:
    label: "Daily pillbox"    # human display name; use this, NOT the key
    slots:
      morning_empty:
        label: "Morning / empty stomach"   # human display name; use this
        products: ["Brand — Substance", ...]   # already sorted casefold
        substances: [...]
      ...
  training:
    label: "Training pillbox"
    slots:
      pre_workout:
        label: "Pre-workout"
        products: [...]
        substances: [...]
      post_workout:
        label: "Post-workout"
        products: [...]
        substances: [...]
benefits: [...]
risks: [...]
warnings: [...]               # list of warning dicts
kept_together: [...]
explanations: {...}
```

Pillbox/slot rendering rules:
- The pillbox dict preserves insertion order from `build_empty_schedule_pillboxes`, which mirrors `pillboxes.yaml` order.
- Slots inside each pillbox preserve insertion order too.
- Both pillbox and slot mappings carry a `label` field — use it. Never display raw keys (`daily`, `morning_empty`).
- `products` is the human-readable list of "Brand — Substance" strings; print these verbatim with a bullet.
- A slot is "empty" iff `len(slot["products"]) == 0`.
- A pillbox is "empty" iff every one of its slots is empty.
