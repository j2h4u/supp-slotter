<objective>
Cut the test suite's dependency on live `data/` content by deleting the LIVE_DATA snapshot/invariant tests and replacing their coverage with a `planner check` gate in CI + justfile. Make cmd_* the testable seam: each cmd_* accepts an optional `data_root` and returns a structured result dataclass instead of being a print-and-`sys.exit` smear. Convert eligible subprocess tests to direct in-process calls against those return values.

Purpose: today, ten tests in test_phase_01/02/schemas assert facts about live data files that planner check already enforces. They are noise — every legit data edit forces a test rewrite. After this plan, `planner check` is the single source of truth for "live data is valid" (enforced in CI), and tests assert on structured cmd_* return values rather than parsing stdout. Net result: faster iteration, no double-bookkeeping, no stringly-typed assertions for command output.
Output: result dataclasses module, refactored cmd_* signatures, CLI adapter in __main__.py, slimmer test_phase_01/02/03/schemas, planner check step in justfile + .github/workflows/test.yml.
</objective>
