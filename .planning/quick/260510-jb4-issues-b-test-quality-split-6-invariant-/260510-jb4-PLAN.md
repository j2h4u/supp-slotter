From planner/cards/warnings.py:
```python
def humanize_warning(
    warning: dict[str, Any],
    *,
    products: dict[str, Product],
    substances: dict[str, Substance],
) -> dict[str, Any]
```

Branches in `_format_warning_entities` (called from `humanize_warning`):
- `warning["product"]` is a string AND in `products` ‚Üí out["product"] = format_product_name(product)
- `warning["product"]` is a string AND NOT in `products` ‚Üí out["product"] = product_id (raw string fallback)
- `warning["type"] == "risk_cluster_load"` AND `warning["cluster"]` is non-empty string ‚Üí out["risk"] = cluster, out["concern"] = cluster
- `warning["type"] == "risk_cluster_load"` AND `warning["active"]` is list of substance ids ‚Üí out["active"] = list of formatted names (with raw-string fallback for unknown ids)
- `warning["substance"]` is a string AND in `substances` ‚Üí out["substance"] = format_substance_name(...)
- `warning["substance"]` is a string AND NOT in `substances` ‚Üí out["substance"] = substance_id
- `warning["source_substance"]` in `substances` ‚Üí out["source"] = format_substance_name(...)
- `warning["source_substance"]` NOT in `substances` ‚Üí out["source"] falls back to `warning["source_name"]` if present, else the id
- Same dual-path for `target_substance` / `target_name`

Branches in `_derive_concern_text`:
- warning_type == "risk_cluster_load" ‚Üí returns "" (sentinel; concern already populated by `_format_warning_entities`)
- non-empty `trait` ‚Üí returns `trait.split(":", 1)[1].replace("_", " ")`
- non-empty `relation` ‚Üí returns `relation.replace("_", " ")`
- otherwise ‚Üí returns `warning_type.replace("_", " ")`

Branches in `humanize_warning` body:
- `warning["action"]` is non-empty string ‚Üí out["action"] = warning["action"]  (overrides `warning_action(...)` lookup)
- `warning["action"]` missing/empty ‚Üí out["action"] = warning_action(warning_type, trait, relation)
- `warning["message"]` is non-string (e.g., dict, None, int) ‚Üí no `note` key emitted
- `warning["message"]` is string AND contains "operator attention" ‚Üí no `note` key (already covered by existing test, do not duplicate)
- `warning["message"]` is string AND does NOT contain "operator attention" ‚Üí `note` key emitted

From planner/engine/review.py ‚Äî `cmd_review_substance(target: str) -> int`:
- target path does not exist on disk ‚Üí returns 1, stderr contains "file not found"
- target path resolves outside `SUBSTANCES_DIR` ‚Üí returns 1, stderr contains "review-substance only accepts paths inside" + display_path of substances dir
- target path has non-`.yaml` suffix ‚Üí returns 1, stderr contains "review-substance only accepts .yaml files"
- `data/traits.yaml` is empty (parses to no traits) ‚Üí returns 1, stderr contains "data/traits.yaml: no traits found"

Existing review-substance test fixtures use `subprocess.run(["uv", "run", "python", "-m", "planner", "review-substance", str(path)], cwd=ROOT, ...)`. For tmp-dir error tests, use the existing `copy_planner_with_data(tmp_path)` helper from `test_phase_03.py` and run with `cwd=tmp_path`.
