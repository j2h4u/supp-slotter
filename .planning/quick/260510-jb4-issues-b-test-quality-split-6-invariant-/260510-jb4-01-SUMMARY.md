## test_review_substance_rejects_empty_traits_file — Branch Determination

The test uses `{}\n` as the traits.yaml content. Schema validation runs (`validate_schemas`) before `load_traits` is called. With an empty mapping `{}`, the YAML schema validator accepts the file as valid (an empty object passes the traits schema), and `load_traits` returns `{}` — an empty dict — which then hits the `if not trait_defs:` guard, emitting `data/traits.yaml: no traits found` to stderr.

**Result: the `load_traits` empty branch is reachable and fires.** The test asserts `"no traits found" in result.stderr or "traits.yaml" in result.stderr` (loose OR for resilience if schema behavior changes). The actual fragment emitted is `data/traits.yaml: no traits found`.
