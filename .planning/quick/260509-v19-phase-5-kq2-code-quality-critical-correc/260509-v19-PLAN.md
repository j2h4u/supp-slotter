**CC-01 — Fix type mismatch in test_inter_product_absorption_relation_blocks_colocation
(tests/test_phase_02.py, around line 640-649):**

The current code:
```python
schedule = run_temp_plan(tmp_path)
zinc_product = fixture_id("prd", "zinc_product")
copper_product = fixture_id("prd", "copper_product")
colocated_pairs = [
    set(slot_entry["products"])
    for slot_entry in flatten_schedule_slots(schedule).values()
    if {zinc_product, copper_product}.issubset(slot_entry["products"])
]

assert colocated_pairs == []
```

The bug: `fixture_id()` returns a raw ID string like `"prd_abc123"`, but
`slot_entry["products"]` contains display strings (product names as returned by
`format_product_name`). The set membership check `{zinc_product, copper_product}.issubset(...)`
can never match — the assertion `colocated_pairs == []` is always true regardless of
whether the planner actually separated the products.

Fix: derive display names from the products directory in `tmp_path`, using the same
`format_product_name(load_product(path))` pattern already used at line 425 in this file.
Look up the product files by scanning `tmp_path / "data" / "products"` for files whose
stems contain the fixture seed name, or construct the path using `fixture_id` + the
product filename convention. The cleanest approach matches what the test already imports:

```python
schedule = run_temp_plan(tmp_path)
products_dir = tmp_path / "data" / "products"
zinc_id = fixture_id("prd", "zinc_product")
copper_id = fixture_id("prd", "copper_product")
zinc_name = format_product_name(load_product(next(products_dir.glob(f"*{zinc_id}*"))))
copper_name = format_product_name(load_product(next(products_dir.glob(f"*{copper_id}*"))))
colocated_pairs = [
    set(slot_entry["products"])
    for slot_entry in flatten_schedule_slots(schedule).values()
    if {zinc_name, copper_name}.issubset(slot_entry["products"])
]

assert colocated_pairs == []
```

`format_product_name` and `load_product` are already imported at line 11. No new imports
needed.
