<objective>
Make `_run_auto_maintenance_unlocked` atomic. Replace the current three independent
mutation phases (normalize_substances → _normalize_card_dir(products) → stacks
rewrite) with a single in-memory `_EditPlan` that is staged to `.tmp` siblings,
then committed via `os.replace`. A failure mid-plan must leave the data directory
byte-identical to its pre-call state.

Purpose: Closes the open CONCERNS.md entry "Auto-maintenance performs multi-file
rewrites without transaction rollback". Today a write failure between steps can
leave normalized substance cards on disk with stale product cards still pointing
at the old refs, or renamed product files with stacks.yaml still naming the old
ids.

Output: Refactored `planner/maintenance.py` (private API + atomic edit plan) and
an updated `tests/test_maintenance.py` (existing failure-path test still asserts
exit 1 + new atomicity test asserts no on-disk mutation when staging fails).
</objective>
