<objective>
Split the 3,435-LOC `planner.py` monolith into a `planner/` package with five
modules — io.py, cards.py, maintenance.py, engine.py, __main__.py — without
changing behavior. This is a lift-and-shift refactor: function bodies move
verbatim, only their module home and import statements change. The original
planner.py is deleted; tests must import from the new submodules and invoke the
CLI as `python -m planner`.

Purpose: Make the codebase navigable. A 3,400-line file is the friction
behind every future feature (review layer, dashboards, doctor extensions). The
five-module split aligns with the SKILL.md / SPEC mental model (I/O,
card-domain, maintenance, scheduling engine, CLI dispatch) and lets
read/edit windows stay focused on one concern at a time.

Output: A working `planner/` package, 48 green tests, byte-identical
`schedule.yaml`, no `planner.py` at repo root.

<critical_gotchas>
1. ROOT path resolution. Today: `ROOT = Path(__file__).parent` where __file__ is
   `<repo>/planner.py`. After move to `planner/io.py` it becomes `<repo>/planner/`.
   Fix: `ROOT = Path(__file__).resolve().parent.parent`. All other path
   constants (DATA_DIR, SCHEMA_DIR, …) are derived from ROOT and will follow.

2. Drop the PEP 723 inline-script-metadata block (lines 1–7 of planner.py).
   pyproject.toml already declares pyyaml + jsonschema. The shebang-style
   header is meaningless inside a package and would just be dead clutter.

3. Lazy jsonschema import. The current `schema_errors` does `import jsonschema`
   inside the function body so importing planner without jsonschema installed
   does not error. Preserve that pattern in `planner/io.py:schema_errors`. Do
   NOT hoist the import to module top.

4. Test subprocess invocations. There are 28 occurrences of
   `["uv", "run", "planner.py", …]` across test_phase_01/02/03. Every one must
   become `["uv", "run", "python", "-m", "planner", …]`. Use sed-style
   targeted replace; do not eyeball.

5. Test `copy_planner_runtime` helpers. test_phase_02.py:47-49 and
   test_phase_03.py:147-152 do `shutil.copy2(ROOT / "planner.py", tmp_path / "planner.py")`.
   Replace with `shutil.copytree(ROOT / "planner", tmp_path / "planner")`. The
   accompanying `shutil.copytree(ROOT / "schema", tmp_path / "schema")` line
   stays. Because subprocess runs use `cwd=tmp_path` and ROOT is computed from
   `__file__`, ROOT inside the subprocess will resolve to `tmp_path/` — exactly
   what those isolated tests want.

6. test_phase_03.py also has a separate `tmp_path` flow (~line 245+ etc.) for
   stacks tests; run grep `planner.py` over the whole file to catch every
   reference, do not assume there are only two.

7. test imports. Currently:
       from planner import format_product_name      # test_phase_02, test_phase_03
       from planner import validate_schemas         # test_schemas
   Change to:
       from planner.cards import format_product_name
       from planner.io import validate_schemas

8. tests/conftest.py stays as-is. It puts the repo root on sys.path so
   `from planner.io import …` and `from planner.cards import …` resolve via
   the package directory the same way `from planner import …` resolved via
   the file. No edit needed.

9. main(). Currently `main()` is defined at line 3367 inside planner.py and
   the `if __name__ == "__main__":` guard is at line 3434. Both move into
   `planner/__main__.py`. The argparse epilog text mentions `uv run planner.py …`
   throughout — update the epilog strings to `uv run python -m planner …` so
   the help output stays accurate. (No test asserts on epilog text, so this is
   cosmetic but correct.)

10. README.md and SKILL.md mention `uv run planner.py …` extensively. They
    are NOT in scope for this refactor (constraints don't list them, tests
    don't read them). Leave them; flag for follow-up in the SUMMARY.

11. Internal name references. After moving check_substances etc. into cards.py,
    every call site inside engine.py / cmd_check must use the new module path
    or import the names. Fastest pattern: at the top of engine.py do
    `from planner.cards import (long, list, of, names)` and
    `from planner.io import (constants, helpers)`. Avoid `from planner.cards import *`
    — explicit imports catch typos at import time.

12. Module-level constants used across modules. Things like
    `WARNING_CATEGORY_LABELS`, `REVIEW_CONTEXTS`, `SCHEDULE_COMMENTS` live in
    io.py per the map above; cards.py and engine.py import them by name.
