# Phase 260508-x91 Plan 01: Split planner.py into a planner Package — Summary

Split the 3,435-LOC `planner.py` monolith into a five-module `planner/` package (io, cards, maintenance, engine, __main__) without behavior change. 48 tests pass; `schedule.yaml` is byte-identical to the pre-refactor snapshot.

## What Shipped

- **`planner/io.py`** (216 LOC): repo-root `ROOT` constant, path constants, schema-name registries, `load_yaml`, `schema_errors` (lazy jsonschema), `validate_schemas`, `dump_schedule_yaml`, plus the `SCHEDULE_COMMENTS` / `WARNING_CATEGORY_LABELS` / `REVIEW_CONTEXTS` data tables
- **`planner/cards.py`** (1,702 LOC): card loaders, slug/id helpers, similarity/find scoring, all `check_*` validators, all `format_*` helpers, dashboard/relation helpers
- **`planner/maintenance.py`** (358 LOC): auto-maintenance lock helpers, normalize_substances, file-rename and ref-rewrite helpers
- **`planner/engine.py`** (1,095 LOC): `cmd_check / cmd_plan / cmd_doctor / cmd_find / cmd_review_substance` plus scheduling internals (slot scoring, must_separate, explain_slot_choice, build_substance_slot_names, collect_orphans)
- **`planner/__main__.py`** (85 LOC): argparse dispatch, epilog text rewritten to `uv run python -m planner …`
- **`planner/__init__.py`** (empty): package marker, no re-exports

The original `planner.py` is deleted.

## Follow-ups (Out of Scope)

1. **README.md / SKILL.md still reference `uv run planner.py …`** — flagged in the plan as out-of-scope. These need a doc-only sweep to switch to `uv run python -m planner …`. Recommend a separate quick task; no functional impact.
2. **`cards.py` is 1,702 LOC** — well above the 800 soft target. Natural further splits if/when this becomes a friction point:
   - `planner/cards/loaders.py` — load_card / load_substance / load_product / load_substance_registry / load_product_registry
   - `planner/cards/relations.py` — the global-relation block (relation_endpoint_*, collect_missing_*, components_have_global_relation, etc.)
   - `planner/cards/dashboards.py` — collect_dashboard_substance_refs, build_dashboard_review, check_dashboards
   - `planner/cards/find.py` — search-score helpers + find_substance_results / find_product_results
   - Leaves card.py with loaders + validators + slug helpers + `format_*` (closer to ~600 LOC)
3. **`engine.py` is 1,095 LOC** — second largest. The cleanest extraction would be `cmd_plan`'s ~530-line body and its nested `search` / `seed_with_greedy_assignment` closures into `planner/scheduler.py`, leaving engine.py with the smaller cmd_* dispatchers + their direct helpers.
